## Overview

The **Smart Medical Assistant** is a graduation project that combines modern NLP techniques with classical and ensemble machine learning models to provide preliminary disease predictions based on user-described symptoms. The system accepts natural language input, extracts medical symptoms, severity levels, and duration cues, then predicts the most likely conditions from a dataset of 30 diseases.

> **Disclaimer:** This system is for educational and research purposes only. It does **not** provide medical diagnosis. Always consult a qualified healthcare professional.

---

## Features

- **Natural Language Symptom Extraction** – Understands free-text symptom descriptions (e.g., *"I have a splitting headache and fever since yesterday"*).
- **Severity & Duration Detection** – Identifies mild, moderate, or severe symptoms and acute/subacute/chronic timeframes.
- **Disease Prediction** – Uses TF-IDF features + symptom metadata fed into multiple ML models (Logistic Regression, Random Forest, SVM, XGBoost, Naive Bayes, Decision Tree).
- **Top-5 Predictions** – Returns ranked disease predictions with confidence scores.
- **Treatment & Precautions** – Enriches predictions with treatment recommendations and precautionary advice.
- **Urgency Classification** – Automatically flags conditions that may require immediate medical attention.
- **RESTful API** – Flask backend with CORS-enabled endpoints for easy frontend integration.
- **Comprehensive Testing** – Unit, integration, and model evaluation tests via pytest.
- **Modular Architecture** – Clean separation between data preprocessing, NLP, ML training, and API layers.

## Installation

See [INSTALLATION.md](INSTALLATION.md) for a detailed, step-by-step guide.

### Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/your-org/smart-medical-assistant.git
cd smart-medical-assistant

# 2. Create a virtual environment
python -m venv venv
source venv/bin/activate   # On Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Run the preprocessing pipeline
python -m src.utils.preprocessing

# 5. Train the ML models
python -m src.ml.train_models

# 6. Start the API
python -m src.api.app
```

The API will be available at `http://localhost:5000`.

## Usage

### 1. Symptom Analysis (NLP only)

```bash
curl -X POST http://localhost:5000/api/analyze \
  -H "Content-Type: application/json" \
  -d '{"text": "I have a severe headache and fever since yesterday"}'
```

### 2. Disease Prediction (NLP + ML)

```bash
curl -X POST http://localhost:5000/api/predict \
  -H "Content-Type: application/json" \
  -d '{"text": "fever, cough, body aches, fatigue, and sore throat", "include_details": true}'
```

### 3. List All Supported Diseases

```bash
curl http://localhost:5000/api/diseases
```

### 4. List All Known Symptoms

```bash
curl http://localhost:5000/api/symptoms
```

### 5. Get Disease Details

```bash
curl http://localhost:5000/api/disease/Flu
```

## API Documentation

| Endpoint | Method | Description | Request Body |
|----------|--------|-------------|--------------|
| `/` | GET | API info & available endpoints | – |
| `/api/health` | GET | Health check & model load status | – |
| `/api/analyze` | POST | Extract symptoms from text | `{"text": "..."}` |
| `/api/predict` | POST | Predict disease from symptoms | `{"text": "...", "include_details": true}` |
| `/api/diseases` | GET | List all supported diseases | – |
| `/api/symptoms` | GET | List all known symptoms | – |
| `/api/disease/<name>` | GET | Details for a specific disease | – |

## Testing

The project includes a comprehensive test suite covering preprocessing, NLP, API integration, and model evaluation.

```bash
# Run all tests with verbose output
python -m pytest tests/ -v

# Run with coverage report
python -m pytest tests/ -v --cov=src --cov-report=term-missing

# Run specific test files
python -m pytest tests/test_preprocessing.py -v
python -m pytest tests/test_nlp.py -v
python -m pytest tests/test_api.py -v
python -m pytest tests/test_models.py -v
```

### Test Coverage Summary

| Test File | Coverage Target |
|-----------|---------------|
| `test_preprocessing.py` | Text cleaning, tokenization, stopwords, lemmatization, dataset preprocessing, TF-IDF |
| `test_nlp.py` | Symptom extraction, severity/duration detection, confidence scoring, recommendations |
| `test_api.py` | Health, analyze, predict, diseases, symptoms, error handlers |
| `test_models.py` | Accuracy threshold, prediction shapes, feature extraction, label encoding |
